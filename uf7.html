1. Fundamentos de criptografía

Definición: la criptografía protege la información mediante algoritmos y claves para evitar accesos no autorizados.

Confidencialidad

Solo los usuarios autorizados pueden leer la información.

Integridad

Garantiza que los datos no han sido modificados.

Autenticación

Permite comprobar quién envía o recibe la información.

No repudio

Impide que alguien niegue haber enviado o firmado algo.

Para memorizar: CIA + N = Confidencialidad, Integridad, Autenticación y No repudio.

2. Criptografía clásica

  • Cifrado César: desplaza letras un número fijo de posiciones.
  • Atbash: invierte el alfabeto. No usa clave.
  • Playfair: cifra por pares de letras usando una matriz 5x5.
  • Enigma: máquina de rotores; cambiaba el cifrado en cada pulsación.
Importante de examen: César y Atbash son muy débiles. Playfair cifra por pares. Enigma fue rota por el equipo de Alan Turing.

3. Cifrado simétrico

Idea clave: usa la misma clave para cifrar y descifrar.

  • Ventaja: es rápido y eficiente.
  • Problema principal: la distribución segura de la clave.
  • AES: estándar actual.
  • DES: obsoleto e inseguro.
1 sola clave rápido problema: envío de clave

4. Cifrado asimétrico

Idea clave: usa un par de claves:

  • Clave pública: se comparte.
  • Clave privada: la guarda el propietario.
Regla básica: lo que se cifra con una clave se descifra con la otra.
  • Para confidencialidad, se cifra con la clave pública del receptor.
  • Para autenticación, se usa la clave privada del emisor.

Algoritmos importantes: RSA, Diffie-Hellman, ECC.

Ojo: Diffie-Hellman sirve para intercambio seguro de claves, no para hash.

5. Funciones hash

Una función hash genera una huella digital de tamaño fijo a partir de unos datos.

  • Sirve para: verificar la integridad.
  • No sirve para: cifrar o descifrar.
  • Irreversible: no se puede reconstruir el original a partir del hash.
  • Determinista: mismo mensaje → mismo hash.
  • Efecto avalancha: un pequeño cambio produce un hash muy distinto.

Algoritmos: MD5 (inseguro), SHA-256 (seguro), bcrypt y Argon2 (muy usados para contraseñas).

Frase de examen: Hash = integridad, no confidencialidad.

6. Firma digital

  • Se calcula el hash del documento.
  • Ese hash se cifra con la clave privada del emisor.
  • El receptor lo comprueba con la clave pública del emisor.
La firma digital garantiza: autenticidad e integridad.
Trampa típica: no se firma todo el documento, se firma el hash del documento.

7. PKI y certificados digitales

  • PKI: infraestructura que gestiona claves y certificados digitales.
  • CA: emite y firma certificados.
  • RA: verifica la identidad del solicitante.
  • VA: comprueba si el certificado sigue siendo válido.
  • Certificado digital: vincula identidad + clave pública.
Importante: un certificado autofirmado sirve para pruebas, pero no genera confianza real.

Trampas típicas de examen

  • Hash no cifra ni descifra.
  • Firma digital no da confidencialidad por sí sola.
  • Confidencialidad ≠ autenticidad.
  • AES es simétrico; RSA es asimétrico.
  • DES ya no es seguro.
  • Atbash no usa clave.
  • Playfair cifra por pares de letras.
  • HTTPS usa SSL/TLS.
  • HMAC = integridad + autenticación usando hash y clave secreta.
  • Diffie-Hellman = intercambio seguro de claves.
