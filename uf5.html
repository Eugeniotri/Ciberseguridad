Base

  • Seguridad = confidencialidad, integridad y disponibilidad.
  • Ataque pasivo = espía. Activo = modifica o daña.
  • Intrusión externa = desde fuera. Interna = desde dentro.

Ataques

  • Phishing = engañar al usuario.
  • Sniffing = capturar tráfico.
  • MitM = interceptar y modificar.
  • DoS = 1 atacante; DDoS = muchos.
  • SQLi = inyectar código en BD.
  • Ransomware = cifra y pide rescate.

Protocolos y acceso

  • HTTPS = web segura.
  • SSH = acceso remoto seguro.
  • Telnet y HTTP = sin cifrado.
  • VPN = túnel seguro en redes no fiables.
  • MFA = defensa clave contra fuerza bruta.

Firewall

  • Filtra tráfico según reglas.
  • ACL = reglas de permitir o bloquear.
  • Stateful = controla el estado de la conexión.
  • WAF = protege aplicaciones web.
  • Logs = registro y detección.
  • DMZ = servidores públicos sin exponer la red interna.

Proxy

  • Proxy = intermediario entre cliente y servidor.
  • Caché = ahorra ancho de banda.
  • Transparente = sin configurar el cliente.
  • Inverso = protege servidores internos.

Herramientas y conceptos

  • Nmap = escaneo de puertos.
  • Wireshark = análisis de paquetes.
  • VLAN = segmentación de red.
  • Zero Trust = no confiar en nadie por defecto.
